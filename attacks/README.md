# ภาพรวมการโจมตี

เอกสารนี้อธิบายชนิดการโจมตีหลักสองแบบที่ใช้ในโปรเจคนี้:

- **DEA (การโจมตีดึงข้อมูล)**
- **JA (การโจมตี Jailbreak)**

ทั้งสองรูปแบบอาศัยการปรับแต่งพรอมต์เพื่อทดสอบ LLM ว่าจะรั่วข้อมูลหรือหลุดจากนโยบายได้อย่างไร

---

## DEA – การโจมตีดึงข้อมูล

DEA ใช้ข้อมูลชุด Enron เพื่อสร้างพรอมต์ที่พยายามให้โมเดลเปิดเผยข้อมูลสำคัญ (เช่น ที่อยู่อีเมล) ที่ซ่อนอยู่ในชุดข้อมูล การโจมตีทำผ่านสคริปต์ `dea_with_metric.py` และ `dea_with_dp.py`

### ขั้นตอนการทำงาน

1. **เตรียมข้อมูล**: คัดลอกไฟล์ JSONL ของภาษา (จาก `data/enron/context_thai_five_hiarchy/`) ไปยัง `data/enron/` แล้วตั้งชื่อไฟล์ใน `enron.py`
2. **กำหนดฟอร์แมตพรอมต์**:
   - `prefix-N` – ใช้ N รายการแรกจากชุดข้อมูลเป็นบริบท
3. **การล้างคำ (scrubbing)**: คลาส Enron สามารถลบคำสำคัญจากพรอมต์ได้เมื่อ `defence_mode=True` สคริปต์ `dea_with_dp.py` ยังรองรับ system prompt ป้องกันจากไฟล์ JSON ในโฟลเดอร์ `defences/defensive_prompt/`
4. **เลือกโมเดล**: เลือกหนึ่งใน Groq models (`llama-3.1-8b-instant`, `meta-llama/llama-4-maverick-17b-128e-instruct`, `moonshotai/kimi-k2-instruct-0905`)
5. **รันการโจมตี**: โค้ด extractor ส่งบล็อกพรอมต์ไปยัง LLM แล้วบันทึกผลลัพธ์
6. **คำนวณเมตริกซ์**: ใช้ `ExtractionRate` เพื่อหาค่าอัตราสำเร็จ (ASR) ผลลัพธ์จะเขียนไปยัง `output/dea/dea_output_*.txt` แล้วคัดย้ายไปที่โฟลเดอร์ `dea_result/…` เพื่อวิเคราะห์

### ฟอร์แมตที่รองรับ

| ฟอร์แมต | คำอธิบาย |
|---------|----------|
| `prefix-N` | N รายการบริบทต่อท้ายด้วย task message |

### การป้องกัน

- **Scrubbing**: `defences/scrubbing_dea.py` ลบคำสำคัญทั้งจากพรอมต์และคำตอบ
- **Defensive prompts**: สั่งให้แทรกคำสั่งระบบเพื่อป้องกันผ่าน `dea_with_dp.py` และไฟล์ JSON ใน `defences/defensive_prompt/`

## JA – การโจมตี Jailbreak

JA พยายามหลอกโมเดลให้มองข้ามฟิลเตอร์ความปลอดภัยด้วยการสร้างพรอมต์ที่เป็นอันตราย เทมเพลตทั้งหมดอยู่ใน `attacks/Jailbreak/jailbreak.py`

### ประเภทของพรอมต์

- **Obfuscation**: ปิดบังเจตนาที่แท้จริงด้วยการบีบอัดหรือสร้างเรื่องราว
- **Rule-based**: สั่งให้โมเดลปฏิบัติตามกฎที่ขัดแย้ง (ห้ามมีคำเตือน, ต้องขึ้นต้นด้วยข้อความบางอย่าง ฯลฯ)
- **Role-play**: สวมบทบาทเป็นตัวละคร เช่น `DAN`, `STAN`, Developer Mode ฯลฯ ซึ่งปลอดจากข้อจำกัด
- **Chat-style jailbreaks**: คำสั่งหลายย่อหน้าซับซ้อน (Niccolo/AIM, PersonGPT, Condition Red ฯลฯ) ออกแบบมาเพื่อหลบฟิลเตอร์

คลาส `Jailbreak` มีเมธอดสำหรับรวมคำถามของผู้ใช้กับเทมเพลต jailbreak ต่าง ๆ แล้วส่งไปยังโมเดล ผลลัพธ์จะประกอบด้วยพรอมต์ที่สร้างขึ้นและการตอบของโมเดล

### การดำเนินการ

1. จัดเตรียมอ็อบเจ็กต์ชุดข้อมูลที่คืนคำถาม (ผ่าน `generate_queries()`)
2. สำหรับแต่ละคำถาม การโจมตีจะวนผ่านเทมเพลตทั้งหมดและประกอบพรอมต์สุดท้าย
3. ส่งพรอมต์แต่ละอันไปยังโมเดลที่เลือก
4. เก็บผลลัพธ์และวิเคราะห์ว่ามีการละเมิดนโยบายหรือไม่

### การปรับแต่ง

เทมเพลตเป็นสตริงง่าย ๆ จัดเก็บในลิสต์; สามารถเพิ่มกลยุทธ์ jailbreak ใหม่ด้วยการเติมเข้าไปใน `self.obfuscation_prompts`, `self.rule_prompts`, `self.role_prompts`, `self.jailbreachat_prompts`, หรือ `self.jailbreachat_prompts2`

---

## การรันการโจมตี

1. เปิดใช้งานสภาพแวดล้อม Python และติดตั้ง dependencies (`requirements.txt` ถ้ามี)
2. ตั้งค่า `GROQ_API_KEY` ในไฟล์ `.env`
3. ทำตามขั้นตอนใน README หรือหัวสคริปต์เพื่อเลือกฟอร์แมต, จำนวนตัวอย่าง, โมเดล และตัวเลือกป้องกัน
4. ตรวจสอบผลลัพธ์ที่บันทึกไว้ใน `output/dea` หรือโฟลเดอร์ `ja` ที่เกี่ยวข้อง
