# error_result

โฟลเดอร์นี้เก็บผลลัพธ์ที่เกิดข้อผิดพลาด (errors) ในการรันโจมตีต่างๆ

## โครงสร้าง
- `dea/` - ผลลัพธ์จากการรัน DEA ที่มีข้อผิดพลาด
- `ja/` - ผลลัพธ์จากการรัน JA ที่มีข้อผิดพลาด

## การใช้งาน
- ใช้เพื่อวิเคราะห์ปัญหาเมื่อการรันไม่สำเร็จหรือโมเดลตอบกลับผิดรูปแบบ
- ไฟล์ในโฟลเดอร์นี้มักเป็นไฟล์ `.txt` หรือไฟล์ log ที่เก็บข้อความ error

## หมายเหตุ
- หากพบ error บ่อย ให้ตรวจสอบ API keys, quota, หรือรูปแบบพรอมต์ที่ส่งไปยังโมเดล
